## 一、项目概览

**Battle Soul** 是一款基于 C# / .NET WinForms 的**回合制卡牌战斗游戏**。玩家通过打出手牌与敌方 AI 进行战斗，融合了**普通攻击、叠层攻击（五指拳心剑）、持续伤害 DOT、格挡、护盾、反击、抽牌、回复能量**等多种机制，并支持**普通模式**与**无尽模式**。

**核心文件：** [FormMain.cs](file:////workspace/FormMain.cs) — 包含 1500+ 行核心战斗逻辑。

### 3. `DrawToHand()` 抽牌逻辑（行 429-451）

```csharp
private void DrawToHand(int count, bool allowOverdraw = false)
{
    for (int i = 0; i < count; i++)
    {
        if (!allowOverdraw && hand.Count >= HandMax) break;
        if (drawPile.Count == 0)  // 抽牌堆空 → 洗入弃牌堆
        {
            if (discardPile.Count > 0)
            {
                drawPile.AddRange(discardPile);
                discardPile.Clear();
                Shuffle(drawPile);
            }
            else break;
        }
        var card = drawPile[0];
        drawPile.RemoveAt(0);
        hand.Add(card);
    }
}
```

**讲解要点：**
- 经典 **"抽牌堆 → 手牌 → 弃牌堆 → 洗回抽牌堆"** 循环。
- `allowOverdraw` 参数允许部分卡牌突破手牌上限（如"无中生有"）。

### 3. ⭐ 重点卡牌：`DoStackAtk()` 五指拳心剑（行 692-753）

这是**全游戏最复杂的卡牌**，包含**指数级伤害增长、牌堆消耗、彩蛋触发**三层逻辑：

```csharp
private void DoStackAtk()
{
    // ① 伤害公式：6 × 3^n（n 为当前叠层数，最多 4 次）
    long dmgLong = 6L * (long)Math.Pow(3, Math.Min(playerStack, 4));
    int dmg = (int)Math.Min(dmgLong, int.MaxValue);
    // 伤害序列：6 → 18 → 54 → 162 → 486（第五次，最大）

    // ② 叠层计数 +1（普通模式限制 5 层，达到后消失）
    playerStack = Math.Min(playerStack + 1, 5);

    DealDamageToEnemy(dmg, "玩家 使用 五指拳心剑");

    // ③ 达到 5 层：从所有牌堆移除这 2 张牌（彻底消失）
    if (!isEndlessMode && playerStack >= 5)
    {
        ExhaustStackCards();

        // ④ 彩蛋：第五次攻击正好击杀敌人 → 播放诗词 + 特殊音乐
        if (beforeHp > 0 && enemyCurHp <= 0)
        {
            var poem = "历经五十四次劫，劫云仍旧漫遮天...";
            var bf = new BroadcastForm(poem, durationMs: 21000);
            isPaused = true;
            axWindowsMediaPlayereasteregg.URL = "Sound\\游戏彩蛋.mp3";
            bf.ShowDialog(this);
        }
    }
}
```

**讲解要点：**
- **指数增长**：6 → 18 → 54 → 162 → 486，最后一击可以击杀满血 Boss。
- **卡牌消耗机制**：使用完 5 次后，这 2 张牌从所有牌堆中消失，模拟"大招用完"。
- **彩蛋触发**：需要**恰好由第五次五指拳心剑击杀敌人**才能触发，增加游戏趣味性。

## 五、核心模块四：伤害结算系统

### `DealDamageToEnemy()` 统一伤害结算（行 1028-1081）

```csharp
private void DealDamageToEnemy(int dmg, string source)
{
    int remaining = dmg;

    // ① 护盾先抵挡（所有伤害优先扣护盾）
    if (enemyShield > 0)
    {
        int used = Math.Min(enemyShield, remaining);
        enemyShield -= used;
        remaining -= used;
    }

    // ② 格挡减伤 30%（向上取整）
    if (enemyIsBlock && remaining > 0)
    {
        int reduced = (int)Math.Ceiling(remaining * 0.3);
        remaining -= reduced;
    }

    // ③ 扣血（含无尽模式溢出伤害传递）
    enemyCurHp -= remaining;
    if (enemyCurHp < 0 && isEndlessMode)
    {
        // 溢出伤害留到敌人下一条命
        enemyOverflowDamage += -enemyCurHp;
        enemyCurHp = 0;
    }

    // ④ 敌方反击姿态：反弹一半伤害给玩家
    if (enemyIsCounter && dmg > 0)
    {
        int reflect = Math.Max(1, dmg / 2);
        playerCurHp -= reflect;
    }
}
```

**讲解要点：**
- **四层防御链**：护盾 → 格挡 → 生命值 → 反击，每层独立判断。
- `DealDamageToPlayer` 是**镜像逻辑**，玩家/敌人使用相同结算流程。

## 六、核心模块五：敌方 AI

### `EnemyAction()` 分段式 AI（行 1308-1379）

敌人行为根据**血量百分比**切换策略，是一种**简单但有效的状态机设计**：

```csharp
private void EnemyAction()
{
    double hpRatio = (double)enemyCurHp / enemyMaxHp;
    double r = rand.NextDouble();   // 0~1 随机数

    if (hpRatio > 0.7)       // ① 高血量段 (>70%)：以普通攻击为主
    {
        // 80% 普通 / 15% 2倍 / 5% 4倍
        if (r < 0.80)  DealDamageToPlayer(enemyAtk,     "敌人 普通攻击");
        else if (r < 0.95) DealDamageToPlayer(enemyAtk*2, "敌人 强化攻击");
        else              DealDamageToPlayer(enemyAtk*4, "敌人 爆发攻击");
    }
    else if (hpRatio > 0.3)  // ② 中血量段 (30%~70%)：DOT 或 护盾
    {
        // 50% DOT（即时4 + 叠层） / 50% 护盾（30 点）
        if (r < 0.5) { DealDamageToPlayer(4, "敌人 DOT侵蚀"); playerDotCount++; }
        else         { enemyShield += ShieldValue * 2; }
    }
    else                     // ③ 低血量段 (<30%)：绝望反扑 或 保命护盾
    {
        // 50% 4倍攻击（临死反扑） / 50% 大护盾
        if (r < 0.5) DealDamageToPlayer(enemyAtk * 4, "敌人 绝望反扑");
        else         enemyShield += ShieldValue * 2;
    }
}
```

**讲解要点：**
- **状态机 + 概率分支**：血量决定策略组，随机数决定具体行动。
- **行为变化明显**：玩家能感知到敌人在不同阶段的威胁差异。

## 七、核心模块六：回合流程控制

### `BtnEndTurn_Click()` 结束回合（行 1256-1305）

```csharp
private void BtnEndTurn_Click(object? sender, EventArgs e)
{
    // ① 手牌超限：让玩家选择弃牌
    if (hand.Count > HandMax) { /* 弹窗选择弃牌 */ }

    // ② 切换为敌方回合
    isplayerTurn = false;

    // ③ 结算：玩家身上的 DOT（敌人之前施加的）
    ApplyPlayerDotTickIfDue();
    CheckGameOver();

    // ④ 敌人行动
    EnemyAction();

    // ⑤ 结算：敌人身上的 DOT（玩家之前施加的）
    ApplyEnemyDotTickIfDue();

    // ⑥ 回合末统一结算（清除 buff、重置能量、切换回玩家回合、抽牌）
    ResolveEndOfRound();
    CheckGameOver();
}
```

### `ResolveEndOfRound()` 回合末统一结算（行 1382-1416）

```csharp
private void ResolveEndOfRound()
{
    playerIsBlock = false;       // 清除所有临时 buff
    playerIsCounter = false;
    enemyIsBlock = false;
    enemyIsCounter = false;

    playerEnergy = 4;             // 重置能量为 4
    enemyEnergy = 4;

    isplayerTurn = true;          // 切换回玩家回合

    if (!isFirstTurn)             // 非首回合：每回合抽 3 张
        DrawToHand(3, allowOverdraw: true);
    else
        isFirstTurn = false;
}
```

**讲解要点：**
- **DOT 的延后结算机制**：`xxxJustApplied` 标志确保新施加的 DOT **不会立即生效**，要到**下一个回合**才开始结算，避免"刚被 DOT 立刻就扣血"的不合理体验。
- **抽牌节奏**：首回合抽 5 张，之后每回合抽 3 张。

## 九、整体架构总结

```
┌────────────────────────────────────────────────────────┐
│              BattleSoul 主窗体 (FormMain.cs)              │
├─────────────┬─────────────┬─────────────┬────────────┤
│  卡牌系统    │  伤害系统    │  回合系统    │  UI 渲染   │
│  - 卡组构建  │  - 护盾扣减  │  - 玩家出牌  │  - 手牌按钮│
│  - 抽/弃牌   │  - 格挡减伤  │  - 敌方行动  │  - 数值标签│
│  - 手牌渲染  │  - DOT 结算  │  - 回合结算  │  - 日志框  │
│  - 效果DoXxx │  - 反击反弹  │  - 结束判定  │  - 全屏    │
└─────────────┴─────────────┴─────────────┴────────────┘
```

**核心设计原则：**
1. **单一职责** — 每个 `DoXxx()` 方法只处理一张卡的效果。
2. **统一入口** — 所有伤害走 `DealDamageToXxx()`，保证护盾/格挡/反击逻辑一致。
3. **状态机思想** — 敌人 AI 根据血量切换策略组，代码简洁易读。
4. **可扩展性** — 新增卡牌只需：加枚举 → 加 `counts` 数量 → 写 `DoXxx()` → 在 `PlayCard()` 中加 `case`。
